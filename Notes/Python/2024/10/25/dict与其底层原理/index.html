    Dict与其底层原理

    本文主要阐述 ✨Dict与其底层原理✨

    dict 是一种内置的数据结构，通常称为字典或映射（Mapping），用于存储键值对（key-value pairs）。字典的键必须是不可变类型，例如字符串、整数或元组，而值可以是任意数据类型。

    主要特点：查找和插入操作的时间复杂度接近 O(1)，这使得字典在需要快速访问数据时非常有用。

    dict 构造约束 引用：

    • dict是由 key 和 value 组成的。
      • 其中 key 规定不能重复。如果字典键重复赋值的话，默认会取最后的赋值。
    • 键必须是可哈希的。大多数Python对象可以作为键，但它们必须是可哈希的对象。像列表和字典这样的可变类型，由于它们不是可哈希的，所以不能作答为键。所有不可变的类型都是可哈希的，因此它们都可以做为字典的键。

    特点¶

    字典的性能特点：

    1. O(1) 时间复杂度：在理想情况下（没有冲突的情况），字典的查找和插入时间复杂度为 O(1)，因此即便字典包含大量数据，查找和插入操作也可以很快完成。
    2. 动态扩容：当字典的存储桶接近装满时，Python 会自动扩容，并重新分配存储桶，以减少哈希冲突的概率。扩容过程会导致字典的性能暂时下降，但可以提高后续操作的效率。

    基本用法¶

    简单介绍下用法

    my_dict = {"name": "Alice", "age": 25}
my_dict["city"] = "New York"  # 添加
my_dict["age"] = 26           # 更新
name = my_dict.get("name")  # 返回 "Alice"
my_dict.pop("age")

    底层原理¶

    Python 字典的底层实现依赖于哈希表，这是一种通过哈希函数来快速查找元素的数据结构。每个键在字典中都会通过哈希函数生成一个哈希值，这个值决定了该键在哈希表中的存储位置。

    哈希表的结构

    1. 存储桶（Bucket）： 哈希表中包含多个存储桶，每个存储桶用于存储键值对。键经过哈希函数转换成一个整数索引，然后映射到某个存储桶中。
    2. 哈希函数： 哈希函数用于将键映射为哈希值。Python的哈希函数通常会对键进行一些数学运算，以确保哈希值的分布均匀。Python 字典中使用的哈希函数可以根据键的类型变化，以适应不同类型的数据。
    3. 冲突处理： 由于哈希表的存储位置有限，不同的键可能会映射到相同的存储位置。这种情况称为“哈希冲突”。Python 字典使用 开放地址法（Open Addressing）来处理冲突，具体方法是二次探测（在冲突发生时，依次检查下一个存储位置，直到找到空位）。常见的有：线性探测、二次探寻、双重探测（具体见数据结构——Hash的知识）

    哈希表的工作流程：

    1. 插入：当插入一个键值对时，Python 字典会计算键的哈希值，然后找到对应的存储位置并存储键值对。如果发生冲突，会采用开放地址法找到下一个可用位置。
    2. 查找：当查找一个键时，Python 字典会通过哈希函数找到其存储位置，直接返回该位置的值。如果该位置没有存储对应的键，则继续探测其他位置。
    3. 删除：当删除一个键值对时，Python 字典会找到对应的位置并删除键值对，同时处理冲突区域的重新分配。

    实际开发的应用¶

    数据库索引、缓存、数据聚合、查找表等，例如：

    1. 缓存：将计算结果存储在字典中，以便快速访问。
    2. 查找表：在需要频繁查找的场景中，可以利用字典的快速访问特性实现查找表。
