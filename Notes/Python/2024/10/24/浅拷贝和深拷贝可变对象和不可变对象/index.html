    浅拷贝和深拷贝、可变对象和不可变对象

    本文主要阐述 ✨可变对象 & 不可变对象✨ 和 ✨浅拷贝 & 深拷贝✨。

    前文阐述了概念与逻辑，后文举出简单的例子。

    在 Python 中，浅拷贝 shallow copy 和 深拷贝 deep copy 是两种复制对象的方法。 在处理可变对象时会有不同的结果。

    可变对象 & 不可变对象

    首先介绍✨可变对象 & 不可变对象✨

    可变对象、不可变对象¶

    不同类型差异出现在内存管理和操作：

    可变对象 Mutable Objects¶

    是指其值可以在原地被修改而不改变对象的内存地址。也就是说，对于一个可变对象，可以直接修改它的内容，而无需创建一个新对象。

    “可变”：在原有内存地址，就可以实现修改

    Python 中的可变对象有：

    • list
    • dict
    • set
    • 可以自定义

    不可变对象 Immutable Objects¶

    是指其值一旦创建就不能修改。如果尝试修改它的值，Python会创建一个新的对象并将这个新的值赋给变量，而不是在原对象上进行修改。

    Python 中的可变对象有：

    • int
    • float
    • str
    • tuple

    “不可变”： 原有对象的内存不能改变，要改变那就新创建一个。

    例如，不可变对象可以用作字典的键，而可变对象不行，因为字典要求键值在其生命周期内不变。

    Note

    python 中 所有变量都是引用。每个变量名都是标签，不直接存储数据，而是贴在对象上作为标识（即指向对象的内存位置）。

    一个对象可以有多个标签即变量名。

    我们可以用 id() 确定对象，一个对象只能有一个 id。

    具体例子，可观看 【【Python】 函数传参：同为变量，为什么列表可变，字符串不可变？】

    不可变对象·引用计数¶

    在python内，不可变对象的内存管理方式采用 引用计数。

    即：python 不会对值相同的不可变对象，申请单独的内存空间，只会记录引用次数。

    浅拷贝、深拷贝¶

    深拷贝、浅拷贝的主要讨论目标为：可变对象。

    关键

    浅拷贝：拷贝了原始元素的内存地址，即创建了新的元素，该元素指向原始元素的内存位置。此时，被拷贝的可变对象改变 -> 拷贝也会改变 / 改变拷贝 -> 改变被拷贝对象 深拷贝：新建了副本，与原始元素、原始内存完全独立。此时，拷贝得到的副本和原始原素互相独立、互不影响
